# Purpose

This document is intended to catalog known edge cases per domain (e.g. ownership transfer mid-planning, RSVP after invitation expiry, contribution refunds, double-booked vendors) so they are handled consistently across mobile, web, and API.

---

# Current State

Edge cases have not yet been consolidated into a standalone document.

Domain PRDs note some edge-relevant rules inline (for example, Occasion PRD's lifecycle transition rules, and Marketplace PRD's booking overlap rules in Business Rules BR-025). As new edge cases are discovered during implementation or QA, they should be added here rather than resolved ad hoc.
