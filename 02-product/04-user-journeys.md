# User Journeys

**Version:** 1.0.0

**Status:** Approved

---

# Purpose

This document defines the end-to-end journeys of each primary persona in Pamora.

A User Journey describes the complete flow from a user's objective to its successful completion.

These journeys guide:

* Product Design
* UX/UI Design
* API Design
* Engineering
* QA Testing
* Product Validation

Every major feature should support at least one documented journey.

---

# Journey 1 — Create an Occasion

## Primary Persona

Occasion Host

## Goal

Create a new Occasion and prepare it for planning.

## Trigger

The Host decides to organize an important occasion.

## Flow

1. Sign in or create an account.
2. Tap **Create Occasion**.
3. Select Occasion Type.
4. Enter basic details.
5. Choose privacy settings.
6. Generate the Occasion workspace.
7. Land on the Occasion Dashboard.

## Success Criteria

* Occasion exists.
* Workspace is ready.
* Host becomes the owner.
* Initial lifecycle status is **Draft**.

# Journey 2 — Build the Planning Team

## Primary Persona

Host

## Goal

Invite trusted people to help organize the Occasion.

## Flow

1. Open Committee.
2. Add members.
3. Assign responsibilities.
4. Send invitations.
5. Members accept invitations.
6. Committee becomes active.

## Success Criteria

Planning responsibilities are distributed among participants.

---

# Journey 3 — Plan the Occasion

## Primary Persona

Committee Member

## Goal

Coordinate planning activities.

## Flow

1. View assigned tasks.
2. Update progress.
3. Create milestones.
4. Record planning decisions.
5. Monitor timeline.
6. Resolve pending work.

## Success Criteria

Planning progresses according to schedule.

---

# Journey 4 — Invite Guests

## Primary Persona

Host

## Goal

Notify guests about the Occasion.

## Flow

1. Open Invitations.
2. Select invitation template.
3. Customize invitation.
4. Generate public invitation page.
5. Share invitation link.
6. Guests receive invitations.

## Success Criteria

Guests can access Occasion information without requiring the mobile application.

---

# Journey 5 — RSVP

## Primary Persona

Guest

## Goal

Respond to an invitation.

## Flow

1. Open invitation.
2. View Occasion details.
3. Select attendance status.
4. Submit RSVP.
5. Receive confirmation.

## Success Criteria

Attendance information updates immediately for organizers.

---

# Journey 6 — Make a Contribution

## Primary Persona

Guest

## Goal

Support the Occasion financially.

## Flow

1. Open public Occasion page.
2. View contribution section.
3. Enter contribution amount.
4. Leave optional message.
5. Submit pledge or payment (depending on platform capabilities).
6. Contribution appears according to privacy settings.

## Success Criteria

Contribution is recorded accurately and reflected in financial summaries.

---

# Journey 7 — Find a Vendor

## Primary Persona

Host

## Goal

Locate an appropriate service provider.

## Flow

1. Open Marketplace.
2. Select category.
3. Apply filters.
4. Compare vendors.
5. Review availability.
6. View ratings.
7. Select preferred vendor.

## Success Criteria

The Host identifies a suitable vendor for the Occasion.

---

# Journey 8 — Book a Vendor

## Primary Persona

Host

## Goal

Secure a vendor for the Occasion.

## Flow

1. Choose service or rental.
2. Review pricing.
3. Select date.
4. Submit booking request.
5. Vendor reviews request.
6. Vendor accepts or declines.
7. Booking status updates.

## Success Criteria

Both parties receive confirmation and the booking becomes part of the Occasion plan.

---

# Journey 9 — Execute the Occasion

## Primary Personas

Host, Committee, Vendors

## Goal

Deliver a successful Occasion.

## Flow

1. Review readiness dashboard.
2. Confirm vendors.
3. Complete remaining tasks.
4. Monitor announcements.
5. Execute timeline.
6. Resolve issues.
7. Mark Occasion as completed.

## Success Criteria

The Occasion is completed successfully and all participants have access to the final record.

---

# Journey 10 — Preserve Memories

## Primary Persona

Host

## Goal

Retain the history of the Occasion.

## Flow

1. Upload photos and videos.
2. Store important documents.
3. Review financial summary.
4. Archive the Occasion.
5. Access historical records later.

## Success Criteria

The Occasion becomes a permanent historical workspace.

6. Publish services.

---

6. Complete booking.
7. Receive verified review.

# Organization Journey

1. Create organization.
2. Invite administrators.
3. Create recurring Occasions.
4. Standardize planning templates.
5. Track organizational reports.

---

# Journey Principles

Every journey should:

* Minimize user effort.
* Avoid unnecessary screens.
* Preserve context.
* Be recoverable if interrupted.
* Support future scalability.
