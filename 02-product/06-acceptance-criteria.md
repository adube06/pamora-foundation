# Purpose

This document is intended to consolidate acceptance criteria across all functional requirements, for use by QA and engineering during release testing.

---

# Current State

Acceptance criteria have not yet been consolidated into a standalone document.

Until then, acceptance criteria are authoritative at the point where each requirement is defined: every FR-XXX entry within the domain [PRD files](prd/) includes its own acceptance criteria.

The [Testing Strategy](../04-engineering/09-testing-strategy.md) defines how acceptance criteria should be verified (unit, feature, integration, end-to-end).
